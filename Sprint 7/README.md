# Resumo

**Modelagem de Dados para Data Warehouse**: Nesse curso eu aprendi sobre Data Warehouse, um repositório central de dados da organização orientado por assunto, não volátil e integrado, cujo objetivo é apoiar a tomada de decisões. Compreendi a diferença entre OLTP (processamento de transações do dia-a-dia usando modelo relacional) e OLAP (processamento analítico para tomada de decisões usando modelo multidimensional), além das operações OLAP como Slice and Dice, Ranking e Drilling. Estudei o processo de construção do DW, que inclui identificação de indicadores, modelagem dimensional, ETL e uso de staging area. Entendi que a modelagem multidimensional é composta por dimensões (informações descritivas como tempo, cliente, produto) e tabelas fato (métricas quantitativas como vendas e faturamento), sendo comum ter mais dimensões que fatos. Aprendi sobre os tipos de dimensões, especialmente Slow Changing Dimensions com controle de versão, a importância das chaves artificiais e hierarquias para operações de drill. Por fim, conheci os modelos Star Schema (dimensões desnormalizadas com alta performance) e SnowFlake (dimensões normalizadas com economia de espaço), sendo o primeiro mais utilizado.

**Amazon Q Developer for Programmers and DevOps AWS AI coding**: Através do curso sobre Amazon Q Developer, adquiri conhecimentos sobre como utilizar adequadamente esta ferramenta de IA para desenvolvimento de software, aprendendo a gerar, documentar, explicar e testar código em questão de segundos através de prompts eficazes. Dominei exemplos práticos de código em Python, compreendendo como aproveitar as capacidades do Amazon Q para escrever código de melhor qualidade e de forma mais rápida. O curso me ensinou técnicas de prompting eficiente que permitem maximizar o potencial da ferramenta, transformando o processo de desenvolvimento através da automação inteligente de tarefas rotineiras como documentação e testes, resultando em maior produtividade e código mais robusto.
